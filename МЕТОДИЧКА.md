# Методичка: Парсер подписчиков → Google Sheets
## For Claude Code (Anthropic)

### Контекст задачи

Продюсерское агентство ведёт ~20 клиентов (медицинские клиники, бьюти-проекты). У каждого клиента есть ссылки на соцсети (Instagram, YouTube, TikTok, VK, Telegram, Facebook, Rutube, OK, Дзен, Pinterest, Likee, Twitter, Snapchat).

**Задача:** Парсер, который ежедневно собирает количество подписчиков по каждой ссылке и пишет в Google Sheets в строчном формате.

### Ключевые требования (прочитай внимательно)

1. **Лимиты запросов — жёстко!**
   - Максимум 2 повторных запроса (retry) при ошибке
   - Задержка минимум 1.5 секунды между запросами
   - Клиент говорит: API "жрёт деньги" — 250 запросов это ~60 центов, так что лишнего не гонять
   - Если первый запрос упал — повторить не более 2 раз, потом записать ошибку

2. **Только Apify API**
   - Никаких прямых парсеров через requests/bs4
   - Каждая платформа → свой Apify actor
   - Список акторов в config.py

3. **Формат данных — строчный**
   ```
   Дата (YYYY-MM-DD) | Клиент | Площадка | Подписчиков
   ```
   Каждый день ~200 строк (20 клиентов × ~10 площадок)
   Данные накапливаются — новая порция дописывается в конец листа

4. **Google Sheets**
   - Таблица: https://docs.google.com/spreadsheets/d/1j98FzWldWKJmnn22QNuTsraMMwhVMRijemsEOif8RI0
   - Лист со ссылками: Лист1 (первая строка — шапка)
   - Результаты пишем в лист "Статистика (raw)" (создать если нет)
   - Ошибки пишем в лист "Ошибки" (создать если нет)

5. **Запуск**
   - Раз в день, утром
   - Парсит данные за "вчера" (дата в строке = дата запуска)
   - Если какая-то ссылка упала — не останавливать всё, записать ошибку и идти дальше

### Алгоритм main.py

```
1. Читаем .env
2. Получаем список проектов из Google Sheets (Лист1)
   - Каждая строка: [Клиент, Ссылка1, Ссылка2, ...]
3. Для каждого клиента → для каждой ссылки:
   a. Определяем платформу по домену URL
   b. Выбираем Apify actor
   c. Запускаем с input (username/url)
   d. Ждём результат, достаём followersCount
   e. Если ошибка → retry (макс 2), потом log_error
4. Собираем все результаты в список:
   [{date, client, platform, followers}, ...]
5. Пишем в лист "Статистика (raw)"
6. Пишем ошибки в лист "Ошибки"
```

### Google Service Account (сделать вручную)

1. Зайти в https://console.cloud.google.com/
2. Создать проект (или выбрать существующий)
3. Включить Google Sheets API
4. Создать Service Account → скачать .json
5. Эту таблицу → Поделиться → добавить email service account как Редактор

### Критично! Что НЕ нужно делать

- ❌ Не парсить напрямую через requests/bs4 — только Apify
- ❌ Не гонять 250+ запросов в минуту — ставь delay 1.5с+
- ❌ Не писать всё в один скрипт — модульная структура обязательна
- ❌ Не хардкодить токены — только .env
- ❌ Не крашить весь парсер из-за одной ошибки — логировать и идти дальше
